# ศรีนครเกมส์ ครั้งที่ 30 — เว็บอัปเดตผลการแข่งขันสด

เว็บไซต์แสดงผล + อัปเดตผลการแข่งขันกีฬาสี ของ **โรงเรียนศรีนครมูลนิธิ**
ธีมสี ม่วง–เหลือง ประจำโรงเรียน ไม่ใช้ไลบรารีภายนอกฝั่งเซิร์ฟเวอร์ (ใช้แค่ Node.js ล้วนๆ)

## โครงสร้างไฟล์

```
srinakhon-games/
├── server.js          เซิร์ฟเวอร์ (Node.js built-in http เท่านั้น ไม่ต้อง npm install)
├── seed.js             สคริปต์สร้างข้อมูลตั้งต้นจากตารางแข่งขัน (รันครั้งเดียว/รีเซ็ตข้อมูล)
├── data/matches.json   ฐานข้อมูลรายการแข่งขันทั้งหมด (ไฟล์ JSON ธรรมดา)
└── public/
    ├── index.html       หน้าแสดงผลหลัก (ทุกคนดูได้)
    ├── admin.html        หน้าแอดมินสำหรับอัปเดตคะแนน (ต้องใส่ PIN)
    ├── css/style.css
    ├── css/admin.css
    └── js/app.js, js/admin.js
```

## วิธีใช้งาน

1. ติดตั้ง [Node.js](https://nodejs.org) เวอร์ชัน 18 ขึ้นไป
2. เปิด terminal ในโฟลเดอร์นี้ แล้วรัน

   ```
   node server.js
   ```

3. เปิดเบราว์เซอร์ไปที่ `http://localhost:3000`
   - หน้าแสดงผลสด: `http://localhost:3000/`
   - หน้าอัปเดตผล (แอดมิน): `http://localhost:3000/admin.html`

**ไม่ต้อง `npm install`** เพราะเซิร์ฟเวอร์ใช้แต่โมดูลในตัวของ Node.js (ไม่มี dependency ภายนอก)

## รหัส PIN สำหรับแอดมิน

ค่าเริ่มต้นคือ `2569`

เปลี่ยนรหัสได้โดยตั้งตัวแปรแวดล้อมก่อนรันเซิร์ฟเวอร์ เช่น

```
ADMIN_PIN=1234 node server.js
```

หรือรันที่พอร์ตอื่น:

```
PORT=8080 node server.js
```

## การอัปเดตผลการแข่งขัน

1. เข้า `admin.html` แล้วใส่รหัส PIN
2. ค้นหา/กรองรายการที่ต้องการด้วยชิพเลือกชนิดกีฬา หรือช่องค้นหา
3. กรอกคะแนนแต่ละสี แล้วกด **บันทึก** — ระบบจะคำนวณผู้ชนะให้อัตโนมัติ
   (แต่ **ยังไม่นับเข้าตารางคะแนนสะสม** จนกว่าจะยืนยันว่าจบแล้ว ดูข้อ 4)
4. **สำคัญ**: เมื่อแข่งจบจริง ให้เลือกที่ช่องสถานะเป็น **"✅ จบแล้ว (ประกาศผล)"** แล้วกดบันทึก
   — ระบบจะถือว่าผลนี้เป็นทางการ และเพิ่งจะนับคะแนนเข้าตาราง "คะแนนสะสมประจำสี" ในตอนนั้น
   ถ้าคะแนนที่กรอกยังไม่ถูกยืนยัน หน้าแสดงผลจะขึ้นข้อความ "ยังไม่ยืนยันผล" กำกับไว้ให้
5. ถ้าอยากกำหนดผู้ชนะเองโดยไม่อิงจากคะแนน (เช่น กรณีชนะจากคะแนนกรรมการ/แพ้ฟาวล์)
   ใช้ช่อง "ผู้ชนะ" เลือกทีมที่ชนะได้โดยตรง

หน้าแสดงผลหลัก (`index.html`) จะดึงข้อมูลใหม่ทุก 20 วินาทีโดยอัตโนมัติ ไม่ต้องรีเฟรชเอง
และจะมีเอฟเฟกต์คอนเฟตตี + แบนเนอร์แจ้งเตือนโผล่ขึ้นมาเองทันทีที่มีการยืนยันผลใหม่

## เพิ่ม / แก้ไข / ลบ รายการแข่งขัน

กด **"＋ เพิ่มรายการแข่งขัน"** ที่หน้าแอดมิน จะเปิดฟอร์มให้กรอกได้ครบ:
ชนิดกีฬา, หมวด/รุ่น (เช่น ม.ต้น, ม.ปลาย), วันที่, เวลา, สถานที่, ชื่อผู้ควบคุม, คู่ที่/รอบ,
หมายเหตุ และเลือกได้ว่าเป็น "การแข่งขัน" (มีคู่แข่ง 2 สี + คะแนน) หรือ "กิจกรรม" (ไม่มีคู่แข่ง เช่น พิธีเปิด)

ทุกรายการที่แสดงในตาราง มีปุ่ม **✎ แก้ไข** และ **🗑 ลบ** ต่อท้ายเสมอ

### ตัวอย่าง: เพิ่ม E-SPORT ที่แข่งแยก ม.ต้น / ม.ปลาย สีละ 1 ทีม

1. กด **"⚙ ชนิดกีฬา"** แล้วเพิ่มชนิดกีฬาใหม่ เช่น "E-SPORT ม.ต้น" และ "E-SPORT ม.ปลาย"
   (หรือใช้ "esport" เดิมก็ได้ แล้วใส่ "ม.ต้น"/"ม.ปลาย" ในช่อง **หมวด/รุ่น** ตอนสร้างแมตช์แทน)
2. กด "＋ เพิ่มรายการแข่งขัน" เลือกชนิดกีฬาที่สร้างไว้ ใส่หมวด ทีม 1 กับทีม 2 (สี) วันเวลาสถานที่
3. ทำซ้ำสำหรับคู่/รุ่นอื่นๆ ตามจำนวนที่ต้องการ

## แชมป์ประจำชนิดกีฬา (ประกาศเอง ไม่คำนวณอัตโนมัติ)

เนื่องจากกติกาตัดสินแชมป์ของแต่ละชนิดกีฬาต่างกัน (ผลต่างประตู, จุดโทษ, ผลการพบกัน ฯลฯ)
ระบบจะ**ไม่เดา**ว่าใครเป็นแชมป์ให้อัตโนมัติ แต่ให้ครูเป็นคนกดยืนยันเองที่หน้า **"⚙ ชนิดกีฬา"**
โดยเลือกสีที่ได้แชมป์ + กำหนดคะแนนที่จะได้ (ค่าเริ่มต้น 5 คะแนน) เมื่อประกาศแล้ว
หน้าแสดงผลหลักจะมีส่วน **"🥇 แชมป์ประจำชนิดกีฬา"** ขึ้นมาเองพร้อมคะแนนรวมแยกตามสี

## ข้อมูลตั้งต้น

ข้อมูลการแข่งขันทั้งหมด (64 รายการ) ถูกนำเข้าไว้แล้วจากเอกสารโปรแกรมการแข่งขัน
หากต้องการล้างผลคะแนนทั้งหมดกลับไปเป็นค่าเริ่มต้น ให้รัน

```
node seed.js
```

**คำเตือน**: คำสั่งนี้จะเขียนทับ `data/matches.json` และล้างคะแนน/สถานะที่บันทึกไว้ทั้งหมด

## ปรับแต่งข้อมูลด้วยตัวเอง

แก้ไขได้โดยตรงที่ `data/matches.json` (เป็น JSON ธรรมดา) หรือแก้ที่ต้นทาง `seed.js`
แล้วรัน `node seed.js` ใหม่ หากต้องการเพิ่มชนิดกีฬาใหม่ ให้เพิ่มใน object `SPORTS`
และเพิ่มแมตช์/กิจกรรมด้วยฟังก์ชัน `match({...})` หรือ `event({...})`

## สถิติผู้เข้าชมเว็บ

หน้าแสดงผลหลักจะนับผู้เข้าชมให้อัตโนมัติ ไม่ต้องตั้งค่าอะไรเพิ่ม:

- ตัวเลข **"ผู้เข้าชมวันนี้"** จะขึ้นในกล่องวันที่ด้านบนสุดของหน้าเว็บ (นับผู้เข้าชมไม่ซ้ำต่อวัน)
- ที่หน้าแอดมิน กดปุ่ม **"📊 สถิติผู้เข้าชม"** เพื่อดูรายละเอียดเพิ่ม: ยอดเข้าชมวันนี้/ทั้งหมด
  ผู้เข้าชมไม่ซ้ำวันนี้/ทั้งหมด และกราฟแท่งย้อนหลัง 14 วัน

การนับใช้รหัสสุ่มที่เก็บไว้ในเบราว์เซอร์ของผู้เข้าชม (ไม่เกี่ยวกับบัญชีผู้ใช้ ไม่เก็บข้อมูลส่วนตัวใดๆ)
ข้อมูลเก็บไว้ที่ `data/stats.json` (เก็บย้อนหลังสูงสุด 90 วัน แล้วลบวันเก่าสุดออกอัตโนมัติ)

## ทำให้เข้าถึงได้จากมือถือ/เครื่องอื่นในวงเดียวกัน

หาค่า IP ของเครื่องที่รันเซิร์ฟเวอร์ (เช่น `192.168.1.10`) แล้วให้อุปกรณ์อื่นเปิด
`http://192.168.1.10:3000` (ต้องอยู่ใน Wi-Fi วงเดียวกัน)
